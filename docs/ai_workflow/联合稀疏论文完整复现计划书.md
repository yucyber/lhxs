- 已完成：项目清理、代码骨架、SIHT/SNIHT/SHTP/SNHTP/SCoSaMP 主线实现、快速配置实验、图表与日志机制。
- 已跑通：`quick` 级别的 strong / step-size / weak / rank-aware 全流程。

## 3. 实施顺序

### 阶段 A：先补“跑 paper-scale 的基础设施”

1. 增加 `profile` 配置解析和输出隔离。
2. 把 weak / step-size / rank-aware / strong 都接入 `--profile`。
3. 把 DCT 归一化做成可配项。

### 阶段 B：补“过程可审计”

1. 新增复现状态审计报告。
2. 日志里写清楚本次跑的是哪个 profile。
3. 在 README 中增加 quick / paper 的运行入口。

### 阶段 C：补“最关键的算法差距”

1. 先增强 `RA-SOMP+MUSIC`。
2. 再为 strong phase 的 exact ARIP 预留插槽。
3. exact 公式没有拿到前，不把 strong phase 写成“严格复现”。

### 阶段 D：最后跑重实验

1. 先跑 `paper` 的 step-size compare。
2. 再跑 `paper` 的 weak phase。
3. 再跑 `paper` 的 rank-aware compare。
4. strong phase 先跑高分辨率 surrogate 曲线，exact 公式补齐后再重跑一次。

## 5. 需要特别注意的风险

- exact ARIP 公式不是调参数能补出来的，它是“资料缺口”，不是“代码缺口”。
- paper-scale 弱相变实验量级明显更大，运行时间会远高于 quick。
- DCT 如果关闭列归一化后曲线大幅波动，不一定是代码错，也可能是数值稳定性和论文实现细节差异。

## 6. 我建议的最佳推进策略

1. 先把 `paper` profile 的 weak / step-size / rank-aware 跑出来。
2. 用审计报告把“已经达到什么程度”讲清楚。
3. 同步去找 [21] 的 exact `L/U` 公式来源，拿到后再收 strong phase。
