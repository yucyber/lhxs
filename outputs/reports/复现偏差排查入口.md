# 复现偏差排查入口

当实验图和原论文趋势对不上时，优先按下面顺序回查，不要直接大改算法。

## 1. 先看最近日志

先打开 `日志/` 中最近 2~3 份步骤记录，确认：

- 当前是哪一步改了代码或配置
- 改动文件有哪些
- 当时是否已经记录了“近似实现 / surrogate / TODO”
- 运行命令和输出文件路径是否与现在一致

## 2. 再看实验配置

- 强相变：`configs/strong_phase.yaml`
- 弱相变：`configs/weak_phase.yaml`
- 步长对照：`configs/step_size_compare.yaml`
- rank-aware 对照：`configs/rank_aware_compare.yaml`
- 算法超参数与停止准则：`configs/algorithms.yaml`

重点检查：

- `run` 里是不是快速实验尺度，而不是论文 `paper_reference` 尺度
- `delta_grid`、`n`、`l`、trial 数、band 宽度是否被改小
- SIHT/SHTP 固定步长、SCoSaMP 候选支撑规模是否和当前想对齐的实验一致

## 3. 再看算法实现文件

- `src/joint_sparse_recovery/algorithms/siht.py`
- `src/joint_sparse_recovery/algorithms/shtp.py`
- `src/joint_sparse_recovery/algorithms/scosamp.py`
- `src/joint_sparse_recovery/algorithms/ra_somp_music.py`

优先排查：

- 行支撑检测是不是按 row-l2 norm 做的
- normalized step size 分子分母是否用的是当前支撑上的梯度
- SHTP/SNHTP/SCoSaMP 投影是否只在候选支撑上做
- RA-SOMP+MUSIC 当前是否只是近似实现，不要把它当严格文献版

## 4. 再看公共模块和数据生成

- `src/joint_sparse_recovery/core/support_ops.py`
- `src/joint_sparse_recovery/core/projections.py`
- `src/joint_sparse_recovery/core/stopping.py`
- `src/joint_sparse_recovery/core/metrics.py`
- `src/joint_sparse_recovery/data/matrix_ensembles.py`
- `src/joint_sparse_recovery/data/signal_models.py`

优先排查：

- Gaussian 矩阵是不是 `N(0,1/m)`
- DCT 是否做了列归一化；如果 DCT 曲线和原文偏差大，先回看这里
- MMV 非零值是不是默认 Rademacher `{−1,+1}`
- 成功判据是不是 `||X_hat-X||_F <= 1e-3`
- 停止准则是不是提前触发过多或过少

## 5. 最后看弱相变拟合状态

打开：

- `outputs/tables/weak_phase_curves.csv`
- `outputs/tables/step_size_compare_curves.csv`
- `outputs/tables/rank_aware_compare_curves.csv`

重点看 `fit_status`：

- `logistic_fit`：正常拟合
- `all_success_fallback` / `all_failure_fallback`：说明当前 k 搜索区间没有真正覆盖 50% 转折点
- `crossing_fallback` / `nearest_half_fallback`：说明 logistic 拟合失败，当前是 fallback 估计

如果 fallback 过多，优先调大 `band_grid_size`、`band_trials`、`max_binary_search_steps`，或者把 `n` 和 `delta_grid` 换回更接近论文尺度。

## 6. 当前已知真实性边界

- strong phase transition 当前用的是 `surrogate_arip_v1`，不是 [21] 的严格闭式 ARIP 边界。
- RA-SOMP+MUSIC 当前是紧凑近似实现，不保证完全等价于文献版 rank-aware 算法。
- 默认 `configs/*` 的 `run` 部分是快速可跑配置，不是论文 n=1024 的完整规模配置。
