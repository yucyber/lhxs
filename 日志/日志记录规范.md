# 日志记录规范

这个文件夹专门记录 AI 每次切换步骤时“做了什么、改了什么、怎么自检、下一步走哪里”，方便后续复现结果异常时回溯定位。

## 文件命名

```text
步骤XX_YYYYMMDD_HHMM_步骤名.md
```

示例：

```text
步骤00_20260404_0305_项目清理与工作流方案.md
步骤01_20260404_1042_实验协议配置固化.md
```

## 每份日志必须包含的字段

```text
# 步骤XX：步骤名

- 时间：
- 本步骤目标：
- 本步骤新增/修改文件：
- 本步骤执行的命令：
- 自检结果：
- 遇到的问题与判断：
- 下一步建议：
```

## 使用规则

- 每完成一个 STEP，先写日志，再进入下一 STEP。
- 如果某一步失败，优先读最近 2~3 份日志，不要直接凭印象改代码。
- 日志里必须诚实记录“哪些是已实现，哪些只是留了接口/TODO”。
- 如果某个实验结果和原论文不一致，日志里要写清楚当时使用的 config、脚本入口、输出文件路径和 stop reason / fit_status。
