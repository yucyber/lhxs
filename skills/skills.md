# 联合稀疏论文复现 AI Skills

适用场景：本科毕设 / 课程论文 / 文献复现，目标论文为 **Greedy Algorithms for Joint Sparse Recovery**，当前草稿为你的复现初稿。

> 工作前提：
> - 你的初稿目前已经覆盖了联合稀疏恢复背景、MMV 模型、Joint OMP / Joint HTP / Joint CoSaMP、Gaussian / DCT / 稀疏信道实验等内容。
> - 目标论文真正的主线是 **SIHT / SNIHT / SHTP / SNHTP / SCoSaMP**、**ARIP 理论保证**、**strong / weak phase transition**，以及与 **RA-SOMP+MUSIC** 的比较。
> - 因此，所有 skill 默认优先服务于“让你的稿子和目标论文对齐”，而不是盲目扩写。

---

## Skill 1：论文复现差距扫描

### 目标
对照“目标论文”和“当前初稿”，给出结构化差距清单，避免后续工作跑偏。

### 输入
- 目标论文 PDF / clean.md
- 当前初稿 docx / md

### 输出
- 已覆盖内容
- 缺失内容
- 偏离内容
- 修正建议
- 优先级（P0 / P1 / P2）
- 10 天补齐计划

### 触发关键词
- 对齐
- 差距
- 偏题
- 还缺什么
- 和原论文比
- 我现在稿子的问题

### 提示词模板
```text
你现在是我的毕设复现审稿人。

目标论文：Greedy Algorithms for Joint Sparse Recovery
我的当前初稿：一篇关于联合稀疏恢复的毕业论文初稿

请你严格对照目标论文和我的初稿，按下面 8 个维度输出差距扫描：
1. 研究问题是否一致
2. 算法集合是否一致
3. 理论分析是否一致
4. 实验协议是否一致
5. 评价指标是否一致
6. 图表是否一致
7. 结论是否一致
8. 论文结构是否一致

输出格式固定为：
A. 已覆盖
B. 缺失
C. 偏离
D. 建议改法
E. 优先级（P0/P1/P2）

特别指出：
- 我的 Joint OMP / Joint HTP / Joint CoSaMP 与原论文的 SIHT / SNIHT / SHTP / SNHTP / SCoSaMP 是否发生了错位
- 哪些内容适合保留为“扩展工作”，哪些内容必须改回原论文主线
- 给我一个 10 天补齐计划
```

## Skill 2：实验协议抽取

### 目标
把论文里分散的实验设置抽取成一份可执行实验协议。

### 输入
- 目标论文 PDF / clean.md

### 输出
- 参数表
- 实验流程
- 成功判据
- 停止准则
- 作图规则
- 不明确项清单
- YAML / JSON 配置草案

### 触发关键词
- 实验怎么做
- 参数怎么设
- 原文实验配置
- 复现协议
- 停止准则
- 成功判据

### 提示词模板
```text
请从目标论文中提取“可执行实验协议”，整理成结构化配置。

要求输出 4 部分：

第1部分：实验总览表
字段包括：
algorithm, matrix_ensemble, n, m, k, l, number_of_trials, stopping_criteria,
success_definition, search_strategy, regression_method, plot_type

第2部分：逐实验拆解
分别整理：
- strong phase transition
- weak phase transition
- fixed vs normalized step size
- 与 RA-SOMP+MUSIC 的比较
- Gaussian 与 DCT 两类矩阵实验

第3部分：不明确项
把论文中没有明确写出的内容单独列出，标注为：
- clearly specified
- implied but not explicit
- missing and needs assumption

第4部分：给我一个可直接落地的 YAML 配置草案
不要编造论文里没有的数字；缺失值请写 unknown，并在旁边给出“推荐默认值”和理由。
```

## Skill 3：复现实验代码共创

### 目标
从“论文描述”落到“能跑的项目骨架”，支持最小可复现实验。

### 输入
- 目标论文实验协议
- 现有代码片段（可选）
- 语言栈要求（Python + NumPy + SciPy）

### 输出
- 项目目录结构
- 核心模块职责
- 函数签名
- 编码顺序
- 单元测试建议
- 最小可跑脚本

### 触发关键词
- 帮我搭代码结构
- 怎么组织项目
- 算法实现框架
- 先写哪些模块
- 复现代码怎么拆

### 提示词模板
```text
请你作为我的科研代码合作者，基于目标论文实验协议，生成一个最小可复现实验项目设计。

要求：
1. 用 Python + NumPy + SciPy
2. 算法至少包括：
   - SIHT
   - SNIHT
   - SHTP
   - SNHTP
   - SCoSaMP
3. 单独封装：
   - DetectSupport
   - Threshold
   - least squares / restricted CG projection
   - Gaussian matrix generator
   - DCT row-subsampling generator
   - row-sparse MMV signal generator
   - stopping criteria
   - success check
   - logistic regression fitting for weak phase transition
4. 输出：
   - 项目目录结构
   - 每个文件的职责
   - 每个核心函数的签名
   - 最容易写错的 10 个点
   - 单元测试建议
5. 代码风格要求科研复现友好，不追求炫技

最后再附一个“先跑通最小实验、再扩展全部实验”的执行顺序。
```

## Skill 4：实验异常诊断

### 目标
当实验结果和论文不一致时，快速定位 bug、协议偏差、数值问题或统计问题。

### 输入
- 结果图
- 参数表
- 日志
- 关键代码
- 与原图的对照说明

### 输出
- Top 5 可疑原因
- 原因解释
- 最小验证实验
- 修复优先级
- 修复后的预期现象

### 触发关键词
- 为什么对不上
- 图不对
- 结果偏低
- 不稳定
- 波动很大
- 复现失败

### 提示词模板
```text
下面是我的实验日志、参数表、部分代码和结果图。
请你扮演“复现实验故障分析师”，不要泛泛而谈，按下面 7 类逐项排查：

1. 算法实现错误
2. 矩阵生成与归一化错误
3. 停止准则不一致
4. 成功判据不一致
5. 实验采样流程不一致
6. 统计方法不一致
7. 数值稳定性或线性代数求解问题

请输出：
A. 最可能的前 5 个原因（按概率排序）
B. 每个原因为什么会导致我现在这种图形/结果
C. 对应的最小验证实验
D. 修复优先级
E. 修复后我应该预期看到什么变化

不要只给概念，请结合我贴出来的结果具体判断。
```

## Skill 5：结果解读与论文成文

### 目标
把实验结果变成可放进论文的学术表达，并明确区分“复现成功 / 部分成功 / 有偏差”。

### 输入
- 图表
- 原论文对应图表
- 你的实验说明

### 输出
- 结果描述版
- 分析讨论版
- 与原论文对照版
- 可直接进正文版

### 触发关键词
- 帮我写实验分析
- 结果怎么表述
- 论文里怎么写
- 和原文怎么对照
- 帮我润色实验小节

### 提示词模板
```text
下面是我的实验结果表、图和原论文对应图。

请你输出 4 个版本的文字：
1. 结果描述版：只陈述现象，不解释原因
2. 分析讨论版：解释为什么会这样，但不夸大
3. 与原论文对照版：明确哪些复现成功，哪些没有完全对上
4. 论文正文版：可以直接放进“实验结果与分析”小节

写作要求：
- 语言像本科毕设，不要写成顶会论文
- 不能把“趋势一致”写成“完全复现”
- 要诚实区分：
  a. 定性一致
  b. 定量接近
  c. 存在偏差
  d. 尚未验证
- 最后附一句“这一结果对后续改进意味着什么”
```

- 老师会问什么
- 怎么回答创新点


## Skill 7：ARIP / 相变理论压缩讲解

### 目标
快速把目标论文里的理论部分压缩成“你能讲清楚、但不失真”的版本。

### 输入
- 目标论文理论章节
- 你当前理解或困惑点

### 输出
- 通俗版解释
- 数学版解释
- 论文表述版
- 答辩黑板版
- 常见误解提醒

### 触发关键词
- ARIP 是什么
- strong/weak phase transition
- 理论怎么讲
- 收敛因子稳定因子
- 表 1 怎么看

### 提示词模板
```text
请把目标论文里的理论部分，特别是以下内容讲清楚：
- ARIP 与 RIP 的关系
- Theorem 1 / Corollary 1 在说什么
- strong phase transition 与 weak phase transition 的区别
- 为什么论文说 worst-case sufficient conditions 很悲观
- 为什么 normalized 版本表现更好

请输出 5 个层次：
1. 小白版（不超过 200 字）
2. 本科论文版
3. 数学定义版
4. 答辩口语版
5. 最容易被老师追问的坑点

要求：
- 不要跳步
- 不要只翻译原文
- 解释“这些理论和实验有什么关系”
```

### 触发关键词

- 这个图怎么画
- 图注怎么写
- 横轴纵轴怎么定义

### 提示词模板
```text
下面是原论文图和我自己的实验数据。
请你帮我完成：
1. 这张图的本质含义
```

## 默认使用顺序

1. **先 Skill 1**：差距扫描，防止偏题。
2. **再 Skill 2**：实验协议抽取，拿到准确配置。
3. **然后 Skill 3**：搭代码结构。
4. **跑实验后用 Skill 4**：定位偏差。
5. **结果出来后用 Skill 5**：写论文分析。
6. **答辩前用 Skill 6 + 7 + 8**：讲理论、讲图、讲贡献。

---

## 使用时的总约束

每次调用 skill 时，建议在提示词末尾追加这句：

```text
请严格区分：
- 原论文明确做过的内容
- 我当前已经完成的内容
- 我计划补做但尚未完成的内容
不要把这三者混写。
```
